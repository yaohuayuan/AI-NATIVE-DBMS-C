Execute only Phase P5: Local providers, native C inference and model-structure route.

This is a research and architecture report.

Do not copy llama.cpp source into the repository.
Do not implement the inference engine.
Do not fine-tune a model.
Do not modify source code or ROADMAP.

Treat these as three distinct levels:

A. Early provider integration:
- Ollama;
- llama.cpp server;
- OpenAI-compatible local service;
- Qwen/GLM and other suitable local text models;
- cloud provider;
- Raspberry Pi remote provider.

B. Long-term native C inference:
- learn from llama.cpp/ggml/GGUF;
- build a smaller C17 runtime owned by aidb;
- initially support one architecture and CPU batch=1;
- validate numerically against reference implementations.

C. Later model research:
- adapters/fine-tuning;
- database-aware input/output heads;
- deeper model structural modifications;
- distillation;
- foundation-model training only as an explicitly unrealistic near-term endpoint.

1. Provider abstraction and capability matrix.

   - tensor/dtype/ownership;
   - operators;
   - GGUF;
