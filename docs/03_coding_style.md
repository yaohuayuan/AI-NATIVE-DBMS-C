# 编码风格

本文约束 AI-NATIVE-DBMS-C 的 C 代码风格。当前 `v0.1.1` 只冻结规范，不新增功能实现文件。

## C 语言标准

- 以 C11 为基础。
- 保持代码可读、可测试、可移植。
- 不使用编译器私有扩展作为核心依赖。
- 如果未来因平台适配需要使用扩展，应封装在 `src/platform` 或局部实现中，并在文档中说明。

## 文件组织

- 公开头文件放在 `include/aidb`。
- 内部头文件放在 `src` 对应模块目录。
- 实现文件放在 `src`。
- 测试放在 `tests`。
- 不提前创建未来模块空实现。

## 函数设计

- 单个函数只做一件事。
- 避免超长函数。
- 复杂流程拆成小函数。
- 不在 `main.c` 中堆业务逻辑。
- 初始化、释放、错误路径要比正常路径更容易被读懂。

## 头文件规则

- 每个头文件必须有 include guard。
- 公开头文件不能依赖内部头文件。
- 内部头文件命名可以使用 `xxx_internal.h`。
- 不把 `*_internal.h` 暴露到 `include/`。
- 公开 API 变更需要同步更新相关文档和测试。

## 注释规则

- 复杂所有权、错误处理、生命周期必须写注释。
- 普通代码不堆废话注释。
- 文档用中文，代码标识符用英文。
- 注释用于解释意图、约束和危险点，不重复代码表面行为。

## 禁止事项

- 不提前创建未来模块空实现。
- 不在业务模块直接散落 `malloc/free`。
- 不在默认测试中联网或调用真实 API。
- 不把 `build/`、`bin/`、`data` 运行产物提交到 Git。
- 不为了让测试通过而删除测试。
