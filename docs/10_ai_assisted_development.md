# AI 辅助开发规范

本文约束 Codex/AI 参与本项目开发时的行为。AI 可以提高迭代速度，但不能替代阶段边界和工程纪律。

## Codex/AI 协作原则

- 每次只做一个小任务。
- 不要一次性创建终局所有 `.c/.h` 文件。
- 不要提前实现未来模块。
- 不要在没有要求时大规模重构。
- 不要破坏绿色构建。
- 不要删除测试来让测试通过。
- 不要默认联网或调用真实 API。
- 不要自动提交 API Key。

## 每次任务应包含

- 当前目标。
- 禁止事项。
- 允许修改的文件范围。
- 验证命令。
- 输出总结要求。

## 失败处理

- 如果构建失败，优先回到最近绿色状态。
- 如果 CMake 缓存卡住，可以由用户手动清理 `build`。
- 不要让 AI 长时间卡在删除目录或权限问题上。
- 遇到权限、网络或环境问题时，应明确说明阻塞点和已验证事实。

## 当前开发节奏

- 先规范。
- 再 core。
- 再 DBMS。
- 再 JSON/AI Runtime。
- 最后 `AI_MATCH`/Optimizer。
